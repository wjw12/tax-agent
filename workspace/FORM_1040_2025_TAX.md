# Form 1040 2025 Tax Computation Guide

Use this guide whenever you need to set `1040.tax_before_credits` or
`1040-SR.tax_before_credits` for tax year 2025.

This file is the source of truth for the standard 2025 Form 1040 line `16`
decision path in this repo. Do not improvise line `16` from memory when this
file applies.

## Canonical Helper

Use the shared helper in `src.federal_income_tax`:

```python
from src.federal_income_tax import compute_form_1040_tax_2025
```

Do not hand-author line `16` with prose arithmetic when the helper covers the
case.

The helper's ordinary-tax path is backed by committed local IRS artifacts under:

- `src/data/irs/2025/form_1040/tax-table.json`
- `src/data/irs/2025/form_1040/tax-computation-worksheet.json`
- `src/data/irs/2025/form_1040/qdcg-worksheet-metadata.json`
- `src/data/irs/2025/form_1040/schedule-d-tax-worksheet-metadata.json`
- `src/data/irs/2025/form_1040/source-manifest.json`

The reproducible parser and verification scripts live in the test repo:

- `scripts/parse_irs_form_1040_tax_2025.py`
- `scripts/verify_irs_form_1040_tax_2025.py`

## Required Inputs

Before calling the helper, compute or collect:

- `filing_status`
- `taxable_income` from Form 1040 line `15`
- `qualified_dividends` from Form 1040 line `3a`
- `capital_gain_distributions` only when `Schedule D` is not required and line
  `7` is only direct capital gain distributions
- if `Schedule D` is present, line `15` and line `16`
- `uses_form_2555`
- `schedule_d_line_18`
- `schedule_d_line_19`
- `has_form_4952_line_4g`
- `requires_schedule_d_tax_worksheet`, derived deterministically from the
  trigger facts above

## Derived Trigger Facts And Enforcement Path

These values are not all user-supplied.

- `schedule_d_line_18` and `schedule_d_line_19` come from the saved
  `Schedule D` payload when `Schedule D` exists.
- `has_form_4952_line_4g` comes from the case facts / payload for `Form 4952`
  when that form is in scope.
- `requires_schedule_d_tax_worksheet` is a derived flag, not a fact to guess
  from prose. The validator derives it in
  `src.input_loader._derive_requires_schedule_d_tax_worksheet(...)`.

Use [src/input_loader.py](../src/input_loader.py) as the
executable reference for these derivations.

Current validator behavior:

- if `has_form_4952_line_4g` is `true`, the case requires the Schedule D Tax
  Worksheet
- otherwise, the trigger turns on only when both `Schedule D` line `15` and
  line `16` are positive and either `schedule_d_line_18` or `schedule_d_line_19`
  is positive
- if the derived trigger is `true`, live validation rejects the standard helper
  path and stops the case

## Form 1040 Line 7 Assembly

Do not describe line `7` loosely as "use Schedule D when it exists."

The exact rule enforced by
`src.input_loader._assemble_form_1040_line_7_amount(...)` is:

- if `Schedule D` exists and `Schedule D` line `16` is negative, use negative
  `Schedule D` line `21` for Form 1040 line `7`
- if `Schedule D` exists and `Schedule D` line `16` is zero or positive, use
  `Schedule D` line `16` for Form 1040 line `7`
- if `Schedule D` does not exist, use `capital_gain_distributions`
- otherwise use `0`

Do not overload `capital_gain_or_loss` as both a Schedule D wire and a
direct-source capital gain distribution field.

When `Schedule D` exists, `capital_gain_distributions` must remain `0` in the
saved 1040 payload. The validator enforces that rule.

## Decision Rule

Use the Qualified Dividends and Capital Gain Tax Worksheet path if any of the
following is true:

- Form 1040 line `3a` is greater than zero
- `Schedule D` is not required and Form 1040 line `7a` includes capital gain
  distributions
- `Schedule D` is present and both line `15` and line `16` are greater than
  zero

Otherwise, use ordinary income tax on line `15`.

## Supported 2025 Scope In This Repo

The shared helper supports:

- ordinary income tax using the official TY2025 tax table when rounded taxable
  income is below `$100,000`
- ordinary income tax using the official TY2025 tax computation worksheet when
  rounded taxable income is `$100,000` or more
- the standard Qualified Dividends and Capital Gain Tax Worksheet path
- 2025 filing statuses supported by the repo's 1040 models

## Escalate Instead Of Guessing

Do not use the standard helper path if any of the following is true:

- the taxpayer is filing `Form 2555`
- the case requires the `Schedule D Tax Worksheet`
- `Schedule D` line `18` or line `19` is nonzero
- `Form 4952` changes the capital-gain tax path
- you cannot tell whether Form 1040 line `7a` is a simple capital gain
  distribution amount or a value that should already be coming from
  `Schedule D`

When one of those applies, stop and route the case through the more specific
worksheet path instead of approximating line `16`.

## Guardrails

- Do not tax all of Form 1040 line `15` at ordinary rates when qualified
  dividends or eligible long-term capital gain are present.
- Do not ignore `Schedule D` line `15` and line `16` when `Schedule D` exists.
- Do not use raw bracket math as the source of truth for TY2025 ordinary Form
  1040 line `16`.
- Do not compute `1040.tax_before_credits` from Form 1040 line `3b`; the
  preferential-rate input is line `3a`.
- Do not treat `1040.tax_before_credits` as processor-derived. It is a
  `computed_input`.
- Keep `tax_before_credits` as a Decimal in JSON payloads, but feed the helper
  the IRS whole-dollar taxable-income amount implied by Form 1040 line `15`.
- Put direct capital gain distributions in `capital_gain_distributions`, not
  in `capital_gain_or_loss`.
- Do not silently leave `schedule_d_line_18`, `schedule_d_line_19`, or
  `has_form_4952_line_4g` unresolved when capital-gain special-rate facts may
  be present.

## Minimal Workflow

1. Compute upstream forms first, especially `Schedule D` when capital gains are
   present.
2. Assemble Form 1040 line `7` with
   `src.input_loader._assemble_form_1040_line_7_amount(...)`, not from a loose
   prose shortcut.
3. Compute Form 1040 taxable income through line `15`.
4. Derive `requires_schedule_d_tax_worksheet` with
   `src.input_loader._derive_requires_schedule_d_tax_worksheet(...)`, using
   `schedule_d_line_18`, `schedule_d_line_19`, `has_form_4952_line_4g`, and
   the Schedule D gain lines.
5. Call `compute_form_1040_tax_2025(...)` with the 2025 inputs.
6. Save the returned amount into `tax_before_credits`.
7. If you are writing an audit sidecar, record that line `16` came from the
   TY2025 Form 1040 helper, backed by the local IRS table / worksheet artifacts.
